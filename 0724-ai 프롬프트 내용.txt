3. (슬라이드 생성)->pdf 첨부
퀴즈 앱 기획서

전체 스타일:
- 그레이스케일 컬러 팔레트 사용
- 각 화면/섹션마다 명확한 라벨(텍스트 박스 형태)부착
- 모바일 화면 가로 폭 기준(375~414px 범위)로 직관적, 사용자 친화적 레이아웃 구성
- 화면 간 이동 흐름은 화상표로 연결하여 내비게이션 경로 명확히 표시

환영 화면(Welcome Screen)
- 화면 중앙에 앱 로고 위치(심플한 아웃라인 형태)
- 로고 아래에 "퀴즈 시작" 큰 버튼(둥근 사각형, 중앙 정렬)
- 하단 또는 버튼 바로 아래에 작은 "게임 방법" 텍스트 링크(언더라인 표시)

카테고리 선택화면(Category Selection)
- 상단 타이틀: "카테고리 선택" 라벨
- 아이콘+텍스트 구성된 퀴즈 카테고리 리스트 또는 2~3열 그리드
- 각 카테고리 아이콘은 단순하고 직관적인 아웃라인 스타일
- 사용자 터치 가능 영역 충분(버튼 스타일)

퀴즈 화면(Quiz Screen)
- 상단: 현재 질문 텍스트 박스(여러 줄 가능, 높이 고정)
- 바로 아래 4개의 객관식 답변 버튼(동일 크기의 세로 나열 또는 2*2 격자 배치)
- 화면 중간 하단에 퀴즈 진행 바(가로막대 스타일, 현재 진행률 표시)
- 가장 하단에 "다음" 버튼 (비활성/활성 상태 표현 가능)

결과 화면(Result Screen)
- 상단 중앙에 전체 점수 숫자 및 레이블(예: "점수:85점")
- 점수 아래에 정답/오답 개수 표시(예: :"정답 8개/오답 2개")
- 하단 중앙에 "다시 하기" 버튼과 "공유" 버튼 나란히 배치, 충분한 터치 영역

사용자 프로필 화면(User Profile Screen)
- 상단에 원형 아바타 자리 표시 심플 윤곽선
- 아바타 오른쪽 혹은 아래에 사용자면 텍스트 라벨
- 사용자 퀴즈 기록(최근 몇 회 점수 리스트 또는 요약) 표시
- 획득한 배지(아이콘 리스트)를 하단에 일렬 또는 그리드 배열

내비게이션 흐름 표시
- 각 화면 간 이동 경로는 굵은 화상표로 표현
- 예: 환영 화면->카테고리 선택->퀴즈 화면->결과 화면->프로필 화면( 및 다시 환영 화면등 복귀 가능성 포함)

만들어 줘.



4. (코드 리뷰)
------------------------------------
1. 화면 해상도 375*812 설정
2. 퀴즈 시작 버튼을 눌렀을 때 퀴즈 상세 화면이 나오도록 구현
3. 보기 중 하나를 선택하면 다음 버튼이 활성화 상태로 바뀌도록 화면 구현해 줘
4. 이전 버튼을 누르면 첫 홈 화면이 나와야 함
앱을 만들어 줘
